Introduction

BlastBot is a self-hosted WhatsApp and Telegram broadcast platform built with Laravel 12 and React (Inertia.js). It lets businesses send bulk campaigns, automate multi-step drip sequences, manage inbound replies in a unified inbox, and generate AI-powered message content — all from a single admin panel installed on your own server.

No third-party subscription fees. No per-message pricing. Full control over your data and infrastructure.

Features Overview

BlastBot includes the following core modules:

Channel Management
  • Connect unlimited WhatsApp Business API accounts and Telegram bots
  • One-click connection verification via the respective APIs
  • Secure credential storage — credentials and bot tokens are hidden from all API responses
  • Channel status tracking: Connected, Disconnected, Error
Contact Management
  • Full CRUD for contacts — create, view, edit, delete
  • Bulk CSV import with automatic header mapping
  • CSV export of all contacts with all fields
  • Organize contacts into lists for targeted sending
  • Tags, country, language, phone, Telegram ID, and email fields
  • Status control: Active, Opted Out, Unsubscribed
  • AI engagement labels: Hot, Warm, Cold, Dormant (auto-assigned by AI classifier)
Template Management (WhatsApp)
  • Create WhatsApp message templates with header, body, footer, and quick reply buttons
  • Submit templates directly to Meta for review via the Graph API
  • Sync approval status in real time: Approved, Pending, Rejected
  • AI compliance score per template (0–100)
  • 13 pre-configured demo templates (WhatsApp + Telegram)
Campaign Broadcasting
  • Send bulk messages to one or multiple contact lists
  • WhatsApp text campaigns and approved template campaigns
  • Telegram campaigns with Markdown formatting
  • Schedule campaigns for future delivery
  • Personalization variables: {{name}}, {{phone}}, {{email}}
  • All sends run through background queue jobs — no browser timeout
Drip Sequence Automation
  • Build multi-step automated message flows
  • Configure delay per step in days and hours
  • Enroll contacts by contact list — duplicate enrollment is prevented automatically
  • Steps fire via background queue jobs every 5 minutes
  • Full step log per contact prevents any message from being sent twice
Unified Inbox
  • All inbound WhatsApp and Telegram replies appear in one inbox
  • Messages received in real time via webhooks
  • New contacts auto-created from first inbound message
  • AI classifies each inbound message and assigns engagement labels
  • Mark single or all messages as read
AI Message Writer
  • Powered by Claude AI (Anthropic)
  • Generates 3 message variants per request
  • Set business type, goal, tone, language, emoji, and CTA preferences
  • Each variant includes compliance score and character count
  • One-click copy to clipboard
AI Campaign Planner
  • Powered by Claude AI (Anthropic)
  • Returns a complete campaign plan: name, overview, estimated reach, open/reply rate forecasts, best send times, step-by-step message sequence, tips, and warnings
  • Inputs: business type, goal, channel, tone, language, duration, audience size, campaign type
Analytics & Reporting
  • Campaign analytics: sent count, failed count per campaign
  • AI logs: every Claude API call with token usage, success/failure, and error messages

Installation Guide

Follow these steps to install BlastBot on your server.

Step 1 — Upload Files

Upload the BlastBot files to your server's web root or a subdirectory. Point your domain's document root to the public/ folder.

/var/www/blastbot/          ← project root
/var/www/blastbot/public/  ← document root (point domain here)
Step 2 — Install Dependencies
composer install --optimize-autoloader --no-dev
npm install && npm run build
Step 3 — Configure Environment

Copy the example environment file and generate the application key:

cp .env.example .env
php artisan key:generate

Open .env and set your database credentials:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=blastbot
DB_USERNAME=your_db_user
DB_PASSWORD=your_db_password

APP_URL=https://yourdomain.com
Step 4 — Run Migrations & Seeders
php artisan migrate --force
php artisan db:seed

The seeder creates the admin account and all demo data (channels, contacts, templates, campaigns, drip sequences, inbox messages, and analytics).

Step 5 — Set Permissions
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache
Step 6 — Default Admin Login
URLhttps://yourdomain.com/admin/dashboard
Emailadmin@blastbot.io
Passwordpassword

Important: Change the default password immediately after your first login via Admin Panel → Profile → Change Password.

Webhook Setup

Webhooks allow BlastBot to receive inbound replies from WhatsApp and Telegram in real time. Your server must be publicly accessible via HTTPS.

WhatsApp Webhook

In your Meta Developer App dashboard:

  1. Go to WhatsApp → Configuration → Webhook
  2. Set the Callback URL to: https://yourdomain.com/webhooks/whatsapp
  3. Set the Verify Token — this must match the value in Settings → WhatsApp Verify Token in BlastBot (default: blastbot_verify)
  4. Subscribe to the messages webhook field
  5. Click Verify and Save

You can change the verify token at any time from Admin Panel → Settings. Make sure the value matches exactly in both Meta and BlastBot.

Telegram Webhook

Telegram webhooks are registered automatically. After connecting a Telegram channel, run:

php artisan telegram:register-webhooks

This command runs hourly via the scheduler. It registers the webhook URL https://yourdomain.com/webhooks/telegram/{channel_id} with Telegram's Bot API for each connected bot.

To register immediately after adding a new Telegram channel, run the command manually once.

Queue & Scheduler Setup

BlastBot relies on Laravel's queue and scheduler for all background processing. Both must be running for campaigns, drip sequences, and inbox classification to work.

Cron Job (Scheduler)

Add this single cron entry to your server (via crontab -e):

* * * * * cd /path/to/blastbot && php artisan schedule:run >> /dev/null 2>&1

The scheduler runs the following commands automatically:

CommandFrequencyPurpose
campaigns:dispatch-scheduledEvery minuteSends scheduled campaigns
drip:dispatch-stepsEvery 5 minutesFires due drip sequence steps
inbox:classify-messagesEvery 2 minutesAI-classifies inbound messages
telegram:register-webhooksHourlyRe-registers Telegram webhooks
contacts:sync-countsDaily 03:00Syncs contact list counts
logs:cleanupDaily 02:00Cleans up old AI logs
Queue Worker (Supervisor — Recommended)

Install Supervisor and create a configuration file at /etc/supervisor/conf.d/blastbot-worker.conf:

[program:blastbot-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /path/to/blastbot/artisan queue:work database --sleep=3 --tries=3 --timeout=60
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=www-data
numprocs=2
redirect_stderr=true
stdout_logfile=/var/log/blastbot-worker.log
stopwaitsecs=3600
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start blastbot-worker:*

After deploying updates, restart the queue workers: php artisan queue:restart

Admin Panel

The BlastBot admin panel gives you full control over all channels, contacts, campaigns, drip sequences, inbox messages, AI tools, and settings from a single interface.

Access the admin panel at: https://yourdomain.com/admin/dashboard

Dashboard

The dashboard gives you a real-time overview of your entire messaging operation.

  • Total Contacts — total number of contacts in the system
  • Active Channels — number of connected WhatsApp and Telegram channels
  • Campaigns Sent — total campaigns dispatched
  • Messages Sent — total individual messages delivered
  • Active Drip Sequences — sequences currently running
  • Unread Inbox Messages — inbound replies awaiting review

Channels

Channels are your WhatsApp Business accounts and Telegram bots. You can connect unlimited channels of each type.

Adding a WhatsApp Channel
  1. Go to Channels → Add WhatsApp Channel
  2. Enter a channel name, your phone number, and your WhatsApp Business API credentials:
    • Access Token — your permanent or temporary Graph API token
    • WABA ID — your WhatsApp Business Account ID
    • Phone Number ID — the ID of the phone number in your WABA
  3. Click Save, then click Verify on the channel to test the connection
Adding a Telegram Channel
  1. Go to Channels → Add Telegram Channel
  2. Enter a channel name, your bot token (from @BotFather), and your bot username
  3. Click Save, then click Verify to confirm the bot is live
  4. After verification, run php artisan telegram:register-webhooks or wait up to 1 hour for the scheduler to register the webhook automatically
Channel Actions
  • Verify — tests the API connection and updates the channel status
  • Disconnect — marks the channel as disconnected without deleting credentials
  • Delete — permanently removes the channel

Contacts

Contacts are the people you message via WhatsApp and Telegram. Each contact can belong to multiple contact lists.

Adding Contacts

You can add contacts in three ways:

  • Manual: Go to Contacts → Add Contact and fill in the form
  • CSV Import: Go to Contacts → Import and upload a CSV file. The first row must be a header row. Supported columns: name, phone, telegram_id, email, country, language
  • Auto-created from Webhook: When a new number sends a WhatsApp or Telegram message, a contact is automatically created
CSV Import Format
name,phone,telegram_id,email,country,language
John Smith,+14155550100,,john@email.com,US,en
Jane Doe,,987654321,jane@email.com,GB,en

Either phone (for WhatsApp) or telegram_id is required per row. Rows missing both are skipped. Duplicate phone numbers or Telegram IDs are also skipped.

Exporting Contacts

Click Export CSV on the Contacts page to download all contacts with all fields as a CSV file.

Contact Lists

Contact lists let you group contacts for targeted campaigns and drip sequence enrollment.

  • Create unlimited lists with a name and optional description
  • Assign contacts to lists during manual creation, CSV import, or contact editing
  • Each list shows a live contact count
  • Lists are used as the target audience when creating campaigns or enrolling in drip sequences

Templates

WhatsApp requires pre-approved message templates for outbound marketing messages. BlastBot lets you create, submit, and manage templates directly from the admin panel.

Creating a Template
  1. Go to Templates → Create Template
  2. Select channel type (WhatsApp or Telegram)
  3. Fill in the template name, language, header, body, footer, and buttons
  4. For WhatsApp, use {{1}}, {{2}} etc. as sequential variables in the body
  5. Click Save as Draft
Submitting a WhatsApp Template for Approval

Once a template is saved as draft, click the Submit button (paper plane icon) on the Templates page. BlastBot sends the template to Meta via the Graph API. The status will update to Pending, and you can click Check Status (clock icon) at any time to sync the latest approval status.

Template Statuses
StatusMeaning
DraftNot yet submitted to Meta
PendingSubmitted, awaiting Meta review (usually 24–48 hours)
ApprovedReady to use in WhatsApp campaigns
RejectedRejected by Meta — review the reason and resubmit

WhatsApp Template Rules: Do not use all-caps words, aggressive language, or excessive punctuation. Use sequential numbered variables ({{1}}, {{2}}). These practices cause rejections.

Campaigns

Campaigns let you send a message to all contacts in one or more contact lists in a single operation.

Creating a Campaign
  1. Go to Campaigns → Create Campaign
  2. Enter a campaign name and select the channel
  3. Select the target contact lists
  4. Choose message type:
    • Text — free-form text message (WhatsApp text or Telegram)
    • Template — WhatsApp approved template (required for marketing messages outside 24h window)
  5. Write your message or select a template. Use {{name}}, {{phone}}, {{email}} for personalization
  6. Set a scheduled date/time or leave blank to send immediately
  7. Click Create Campaign
How Campaigns are Sent

The campaigns:dispatch-scheduled command runs every minute. When it finds a campaign that is scheduled and due, it dispatches a SendCampaignJob queue job for each contact in the target lists. Jobs run in the background with 3 automatic retries on failure.

Campaign Statuses
StatusMeaning
DraftCreated but not yet scheduled
ScheduledWaiting to be dispatched at the set time
SendingCurrently being dispatched by queue workers
SentAll jobs dispatched
FailedDispatch encountered an error

Drip Sequences

Drip sequences automate a series of messages sent to contacts over time. Each step fires at a precise delay after the contact was enrolled.

Creating a Drip Sequence
  1. Go to Drip Sequences → Create Sequence
  2. Enter a sequence name, optional description, and select the channel
  3. Add steps. Each step requires:
    • Step Name — a label for this step (e.g. "Day 1 Welcome")
    • Delay — days and hours after enrollment when this step fires
    • Message Body — the message text. Supports {{name}}, {{phone}}, {{email}}
    • Template (optional) — link an approved WhatsApp template
  4. Click Save Sequence
Enrolling Contacts
  1. On the Drip Sequences list, click the Enroll button (person+ icon) next to a sequence
  2. Select one or more contact lists from the modal
  3. Click Enroll Contacts

Only active contacts who are not already enrolled in that sequence will be added. The system shows the number of newly enrolled contacts.

How Drip Steps are Sent

The drip:dispatch-steps command runs every 5 minutes. For each enrolled contact, it checks each step's delay — if the time since enrollment is greater than or equal to the step's delay, and the step has not already been sent to that contact, it dispatches a SendDripStepJob. Each step is sent at most once per contact.

Example: 3-Step Onboarding Sequence
StepDelayMessage
1 — Welcome0 days, 0 hoursWelcome message sent immediately on enrollment
2 — Follow Up1 day, 0 hoursSent 24 hours after enrollment
3 — Offer3 days, 0 hoursSent 72 hours after enrollment

Inbox

The inbox collects all inbound replies from your WhatsApp and Telegram contacts in one place.

How Inbound Messages are Received
  • WhatsApp: When a contact replies to a WhatsApp message, Meta sends a webhook POST to https://yourdomain.com/webhooks/whatsapp. BlastBot processes the payload and stores the message.
  • Telegram: When a contact messages your bot, Telegram sends a webhook POST to https://yourdomain.com/webhooks/telegram/{channel_id}. BlastBot processes and stores the message.

If the sender is not already a contact, a new contact is automatically created from the incoming message data.

AI Classification

The inbox:classify-messages command (runs every 2 minutes) uses Claude AI to analyze each unclassified inbound message and update the contact's AI Engagement Label to one of: Hot, Warm, Cold, or Dormant. This helps you prioritize follow-ups.

Inbox Actions
  • Click any message to mark it as read
  • Click Mark All Read to clear all unread messages at once

AI Message Writer

The AI Message Writer uses Claude AI to generate 3 high-converting message variants based on your inputs.

How to Use
  1. Go to AI Tools → Message Writer
  2. Fill in the generation form:
    • Business Type — what your business does (e.g. "Online clothing store")
    • Goal — what you want the message to achieve (e.g. "Promote 30% off sale this weekend")
    • Tone — Friendly, Professional, Urgent, Casual, or Formal
    • Channel — WhatsApp or Telegram
    • Language — the language for the message
    • Include Emoji — toggle on/off
    • Include Call-to-Action — toggle on/off
  3. Click Generate Messages
  4. Three variants are returned. Click Copy on any variant to copy it to your clipboard
Compliance Score

Each variant includes a compliance score (0–100) indicating how likely it is to pass WhatsApp's template review policies. Aim for 90+ for WhatsApp templates.

AI Campaign Planner

The AI Campaign Planner generates a complete, structured campaign plan based on your business context and goals.

How to Use
  1. Go to AI Tools → Campaign Planner
  2. Fill in the planning form:
    • Business Type — a short description of your business
    • Campaign Goal — what you want to achieve (up to 500 characters)
    • Channel — WhatsApp or Telegram
    • Campaign Type — Drip, Broadcast, or Mixed
    • Duration — campaign length in days (1–90)
    • Audience Size — estimated number of contacts
    • Tone and Language
  3. Click Generate Plan
What You Get
  • Campaign name and overview
  • Estimated reach, expected open rate, and expected reply rate
  • Best send times
  • Step-by-step message sequence with objectives and send times per step
  • Campaign tips and warnings

Analytics

Campaign Analytics

Go to Analytics → Campaigns to view delivery metrics for all campaigns — sent count, failed count, channel type, and scheduled time.

AI Logs

Go to Analytics → AI Logs to review every Claude API call made by the system:

  • Feature (Message Writer or Campaign Planner)
  • Model used
  • Input and output token counts
  • Success or failure status
  • Error message (if failed)
  • Timestamp

This allows you to track AI costs and debug any generation issues.

Update Profile Information

Change your admin name and email address. Click Save Changes to apply.

Change Password

Help & Support

If you need help with BlastBot, please use the following support channels:

  • CodeCanyon Comments: Post your question on the item page comments for community support
  • CodeCanyon Support Tab: Submit a private support ticket via the item's Support tab on CodeCanyon
Before Contacting Support

Please check the following first:

  • Verify your .env file is correctly configured
  • Ensure the queue worker and cron scheduler are both running
  • Check storage/logs/laravel.log for error messages
  • For WhatsApp webhook issues, confirm the verify token matches exactly in both Meta and BlastBot Settings
  • For Telegram webhook issues, run php artisan telegram:register-webhooks manually
  • For AI tool issues, verify the Anthropic API key is set and has sufficient credits in Settings
