────────────────────────────────────────────────────────────────────────────────
WHY THIS ITEM IS UNIQUE ON CODECANYON
────────────────────────────────────────────────────────────────────────────────

Most WhatsApp/Telegram tools on CodeCanyon are simple bulk senders. BlastBot
goes significantly further with three features that do not exist together in any
current CodeCanyon listing:

1. AI-POWERED INBOX WITH AUTO-CLASSIFICATION
   All inbound WhatsApp and Telegram replies flow into a unified inbox via live
   webhooks. An Artisan command (inbox:classify-messages, runs every 2 minutes)
   uses Claude AI to automatically classify each message and tag contacts with
   AI engagement labels: Hot, Warm, Cold, or Dormant. This turns a passive inbox
   into an intelligent lead-scoring system — a feature that does not exist in
   any other CodeCanyon messaging item.

2. MULTI-STEP DRIP SEQUENCES WITH PRECISE DELAY CONTROL
   Unlike simple "schedule one message" features found elsewhere, BlastBot
   includes a full drip automation engine. Each sequence can have unlimited
   steps, each with independent day + hour delay settings. Contacts are enrolled
   by contact list, duplicate enrollment is prevented, and each step fires via
   a background queue job (drip:dispatch-steps, every 5 minutes) — completely
   automated with no browser dependency. This is a CRM-level drip feature
   normally only found in expensive SaaS tools like ActiveCampaign or
   ManyChat.

3. DUAL AI TOOLS POWERED BY CLAUDE (ANTHROPIC)
   - AI Message Writer: generates 3 channel-optimized message variants with
     compliance scoring, tone control, emoji/CTA toggles, and character counts.
   - AI Campaign Planner: takes a business type and goal and returns a complete
     campaign plan — name, overview, estimated reach, open/reply rate forecasts,
     best send times, a step-by-step message sequence, tips, and warnings.
   Both tools log every API call with token usage and success rates for full
   transparency and cost tracking.

────────────────────────────────────────────────────────────────────────────────
TECHNICAL QUALITY NOTES FOR REVIEWER
────────────────────────────────────────────────────────────────────────────────

- Built on Laravel 12 + React 18 (Inertia.js) — modern, well-supported stack.
- All campaign and drip sends run through Laravel Queue jobs (database driver)
  with 3 auto-retries and 60s timeout — no browser timeouts, no lost sends.
- WhatsApp webhook uses proper hub.verify_token verification. Telegram webhook
  uses Laravel route model binding for clean 404 handling.
- CSRF is properly excluded for webhook routes via validateCsrfTokens(except).
- XSS protection, security headers, and role middleware are applied globally
  to all admin routes.
- Sensitive fields (credentials, bot_token) are hidden on the Channel model.
- All destructive operations (delete contact, delete campaign, etc.) use
  database soft deletes or confirmation modals.
- Full demo seed data is included: channels, contacts, contact lists, templates,
  campaigns, drip sequences, inbox messages, campaign analytics, and AI logs —
  so reviewers can evaluate all features immediately after php artisan db:seed.
